# CAD Model Analyzer Roadmap

这个 Roadmap 不是为了把项目包装成完整商业 CAD 软件，而是用于展示后续工程演进思路：从单文件分析工具，逐步扩展为可集成的 CAD 数据分析模块。

---

## 已完成

- STEP 文件读取：基于 OpenCASCADE `STEPControl_Reader`。
- B-Rep 拓扑统计：Vertex / Edge / Wire / Face / Shell / Solid / Compound。
- 几何分类：常见曲线和曲面类型识别。
- 工程指标：bounding box、表面积、体积、质心。
- 基础模型质量判断：自由边、非流形边、闭合实体候选判断。
- JSON 报告：包含 metadata、topology、curves、surfaces、metrics、quality。
- 批量分析：`--batch <dir> -o <output_dir>` 输出每个模型 JSON 和 `summary.csv`。
- Windows 构建、运行和回归测试脚本。

---

## 下一阶段：更贴近 CAD/CAE 岗位要求

### 1. 模型质量检查增强

目标：从“能统计”提升到“能判断模型是否可用于后续加工、仿真或入库”。

计划增加：

- 空模型检查。
- 多 Solid 检查。
- Shell-only 模型检查。
- 开放边详细列表。
- 非流形边详细列表。
- 零体积或异常体积检查。
- 面/边数量复杂度等级评估。

对应面试能力：

- B-Rep 拓扑理解。
- CAD 数据清洗和质量控制。
- 面向工业软件场景的问题建模能力。

### 2. 批量分析和汇总

状态：基础版本已完成。

已支持：

- `--batch <dir>` 扫描目录下的 STEP/STP 文件。
- 每个模型输出单独 JSON。
- 生成 `summary.csv`，包含文件名、实体数、面数、边数、自由边、非流形边、体积、面积、耗时和报告文件。
- 对异常模型继续处理并记录错误，不中断整批任务。

后续可增强：

- 递归扫描子目录。
- 输出复杂度等级和质量状态摘要。
- 支持 CSV 字段配置。
- 增加大批量任务的进度输出。

对应面试能力：

- C++ 文件系统和异常处理。
- 工业数据批处理流程。
- 工程工具稳定性设计。

### 3. 更多 CAD 格式支持

目标：展示对 CAD 数据交换格式的扩展能力。

计划增加：

- IGES 读取。
- BREP 读取。
- STL 网格输入的基础统计。

对应面试能力：

- CAD 数据转换。
- 几何内核 API 学习能力。
- 多格式输入抽象设计。

### 4. 可视化和服务化

目标：让当前 C++ 分析核心可以被上层产品集成。

计划方向：

- 导出模型缩略图或简单预览数据。
- 封装 HTTP API：上传 STEP，返回 JSON 分析报告。
- 增加简单 Web 页面展示 topology / metrics / quality。
- 后续可接入数据库，形成 CAD 模型资产索引。

对应面试能力：

- C++ 核心模块与上层应用集成。
- 工业软件产品化思维。
- 后端服务和数据结构化能力。

---

## 暂不做的范围

- 不实现完整 CAD 建模器。
- 不实现复杂布尔运算 UI。
- 不做大型三维渲染引擎。
- 不替代商业 CAD 软件。

当前项目重点是：**围绕 OpenCASCADE 展示 CAD 数据解析、B-Rep 分析、质量判断和 C++ 工程化能力**。
